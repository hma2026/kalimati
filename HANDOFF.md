## 0) قاعدة العمل مع Claude (مهم جداً — اقرأها أولاً)
- **اللغة:** كل الردود بالعربية، موجزة، بلا مقدمات.
- **التنفيذ لا التقرير:** نفّذ مباشرة وسلّم ZIP + لقطات تحقّق. لا تكتفِ بتقرير ولا تطلب أذونات لكل خطوة.
- **التحقّق البصري إلزامي:** يوجد متصفّح headless (انظر §7). بعد أي تغيير بصري، صيّر الشاشة فعلياً والتقط لقطة وقارنها بالمرجع. **لا تقل «نظيف/مطابق» إلا بعد رؤية اللقطة.**
- **ممنوع منعاً باتاً:** emoji في الواجهة، صور بديلة/placeholder، ألوان مخترَعة، crop مشوّه، أو تسليم نفس النسخة دون تغيير.
- **لا تكسر المنطق:** محرّك اللهجات، التسجيل الصوتي، التنقّل، التقارير، PWA — تبقى سليمة. عدّل الحقول البصرية/الزخرفية فقط.

## 1) ما هو التطبيق
**كلمة كلمة**: تطبيق PWA عربي للتواصل المعزّز (AAC) وتدريب النطق للأطفال ذوي التأخّر اللغوي/التوحّد. iPad-first، يدعم RTL، أزرار كبيرة، ألوان هادئة.

**المستويات والشاشات:**
- البداية (Splash) ← اختيار الطفل ← الرئيسية (شبكة أقسام).
- المستوى 1: كلمات مفردة. المستوى 2: جمل من كلمتين (أبغى + كلمة). المستوى 3: جمل أطول. المستوى 4: المشاعر والاحتياجات (تبويبات: مشاعر/ألم/تهدئة). المستوى 5: ألوان وأشكال (تبويبات: ألوان/أشكال/تدريب الجملة). المستوى 6: عبارات يومية (تبويبات: تحية/وداع/أكل/طلب/ردود).
- حيوانات وأصوات + تفاصيل الحيوان. ألعاب تعليمية (مطابقة ألوان/حيوانات). حرف الشين. التقرير. الإعدادات (خلف بوابة وليّ الأمر بضغط مطوّل). التهدئة.

## 2) التقنية والبنية
- **Stack:** React 18.3.1 + Vite 5.4.11 + TypeScript 5.6.3 + Zustand + vite-plugin-pwa. Node v22. **لا Tailwind** — CSS يدوي.
- **التصميم:** `src/styles/tokens.css` (رموز التصميم) + `src/index.css` (أنماط عامة، ~1600+ سطر؛ كتل الإصدارات مُلحقة في نهايته).
- **الحالة:** Zustand — `src/store/useNavStore.ts` (التنقّل) + متجر الإعدادات. التخزين: localStorage.
- **النصوص:** محرّك اللهجات `src/data/dialects.ts` هو **مصدر النصّ الوحيد** — لا تكتب نصوصاً ثابتة بديلة.
- **النشر:** Claude **لا يستطيع** النشر. المستخدم ينشر عبر GitHub Desktop → Vercel، ويمسح الكاش. GitHub: `hma2026/kalimati`. Live: `kalimati.vercel.app`.

**أوامر البناء (يجب أن تنجح كلها قبل أي تسليم):**
```bash
npm install
npm run check:assets   # coverage_check.mjs + check_no_emoji_ui.mjs
npm run typecheck      # tsc
npm run build          # tsc + vite build
```

**أمر التحزيم:**
```bash
rm -rf dist dev-dist && zip -0 -r /mnt/user-data/outputs/<name>.zip . \
  -x "node_modules/*" -x ".git/*" -x "dist/*" -x "dev-dist/*"
```
ثم `present_files`. استخدم `python3` (وليس `py`). المخرجات في `/mnt/user-data/outputs/`.

## 3) نظام الأصول (مهم لفهم كيف تُستخدم الصور)
- كل الأصول في `src/assets/images/<category>/<key>.(png|svg)`.
- **`src/assets/assetRegistry.ts`** يفهرس بالاسم (basename بلا امتداد) و**يُفضّل PNG على SVG** عند وجود الاثنين بنفس المفتاح. يستثني مجلدات `ui/` و`avatars/` و`rewards/`.
- **`src/components/Media.tsx`** يحلّ الصورة هكذا (سطر ~41):
  ```js
  const url = getAsset(k) ?? getMediaImage(k) ?? getAssetUrl(k)
  ```
  أي: ضَع `images/<cat>/<key>.png` بنفس اسم مفتاح موجود → يُستخدم تلقائياً، بلا أي تعديل كود.
- `mediaVisual(key, size)` يرسم الصورة. `AssetIcon refKey="..."` يرسم بالاسم.

## 6) أنبوب استخراج الصور (الطريقة المُثبَتة — Python + Pillow)
الفكرة: قص الخلية من الـsheet → إزالة الخلفية البيضاء بـ flood-fill من الحواف (يحفظ الأبيض الداخلي كبطن البقرة) → الإبقاء على أكبر مكوّن متّصل (يحذف النصوص والشظايا) → إزالة الجزر الصغيرة → إزالة الهالة البيضاء على الحواف → تشذيب → PNG شفّاف. للوجوه/الأجسام الفاتحة: **عتبة منخفضة** (لئلا تأكل الفاتح). للهالات العنيدة: **تآكل حافي 2-3px** أو إزالة الأبيض النقي (min≥210) عالمياً إن لم يكن في الجسم أبيض. **تحقّق دائماً بخلفية شطرنجية + رؤية montage قبل القول «نظيف».**

السكربت الكامل مرفق في `ASSET_EXTRACTION_PIPELINE.py` داخل الحزمة.

**مواضع القص المعايَرة (إحداثيات أصلية):**

## 7) المتصفّح الـheadless (للتحقّق البصري — أعِد إنشاءه في كل جلسة)
البيئة تُعاد بين الجلسات. لإعادة التثبيت:
```bash
cd /tmp && npm i @sparticuz/chromium puppeteer-core
```
الإطلاق (انتبه `.default`):
```js
import chromium from '@sparticuz/chromium'   // أو require(...).default
import puppeteer from 'puppeteer-core'
const exe = await chromium.executablePath()
const b = await puppeteer.launch({ args:[...chromium.args,'--no-sandbox','--disable-dev-shm-usage'], executablePath:exe, headless:'shell' })
```
خدمة التطبيق: `cd dist && python3 -m http.server 5055`.
**التنقّل (مُثبَت):** goto `http://localhost:5055/?b=<ts>` → انتظر 1300ms → انقر زر «ابدأ» → انقر زر يحوي «أحمد» → الرئيسية. للوصول لمستوى: انقر زر يحوي اسمه (مثل «المستوى الرابع»). للرجوع: انقر «الرئيسية». **انقر أزراراً حقيقية (button/a)، لا div.** عطّل الكاش `page.setCacheEnabled(false)`. iPad VP 1024×1366، جوال 390×844. خطوط Google لا تُحمَّل headless (بديل النظام — لا يؤثّر على البنية/الألوان).
السكربت الكامل مرفق في `SCREENSHOT_HARNESS.mjs`.

## 8) قرارات تصميم مُثبتة (مطابِقة للمرجع — تحقّقت منها بصرياً)
- **ترويسات المستويات:** L2/L3 = **بانر مصمت** (نص أبيض، خلفية لون المستوى). L4/L5/L6 + الحيوانات/الألعاب = **ترويسة فاتحة** (نص بلون المستوى/داكن على خلفية فاتحة، بطاقتا الرئيسية/النجوم بيضاوان، بلا بانر). [`LevelHeader.tsx` يميّز بـ `level.id`؛ `SectionHeader.tsx` فاتح دائماً.]
- **آبار الصور:** الصورة على بطاقة بيضاء (بلا مربع رمادي). [`.wcard__media,.pcard__visual{background:transparent}` في نهاية index.css.]
- **خلفيات الصفحات:** شبه بيضاء (المرجع ≈#FDFDFD)؛ تينت خفيف جداً لكل مستوى (soft في `levelMeta.ts`).
- **ألوان مستخرَجة بالبكسل:** L4 accent #F15B78, L5 accent #1477DD (L2 #7B3FF2, L3 #2F9B5F, L6 #D99A1E)؛ `--color-bg #F7F5FD`.
- **شاشة البداية الجديدة:** hero واحد (عنوان+طفلان+زخارف) + زرّان بالكود + درع/نص أمان. [`SplashScreen.tsx` + كتلة `.splash` في index.css.]
