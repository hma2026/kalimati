# كلمتي — أداة تواصل ونطق (PWA)

أداة مساعدة للتواصل والتدريب على النطق للأطفال (التوحّد / تأخّر النطق).
**أداة مساعدة للتواصل والتدريب على النطق، ولا تغني عن المختصين.**

تطبيق ويب تقدّمي (PWA): يُثبَّت على الجوال والتابلت والحاسوب، ويعمل **بدون إنترنت** بعد أول فتح، بواجهة عربية كاملة (RTL) مصمّمة للتابلت أولاً.

---

## التشغيل

```bash
npm install        # تثبيت الحزم
npm run dev        # تشغيل بيئة التطوير (مع Service Worker تجريبي)
npm run build      # بناء نسخة الإنتاج إلى مجلد dist/
npm run preview    # معاينة نسخة الإنتاج محلياً
```

المتطلبات: Node.js 18+.

---

## النشر (Vercel)

المشروع جاهز للنشر على Vercel مباشرة (يوجد `vercel.json`):

- Framework: **Vite**
- Build: `npm run build`
- Output: `dist`

يُفضّل اختبار التثبيت و«العمل بدون إنترنت» على نطاق **HTTPS** (Service Worker لا يعمل على http عدا localhost).

---

## مميزات الـ PWA المطبّقة

- **قابل للتثبيت** على Android / iPhone / iPad / Desktop (بطاقة «تثبيت» تلقائية، وإرشادات «إضافة إلى الشاشة الرئيسية» على iOS).
- **يعمل بدون إنترنت** بعد أول تحميل (Service Worker + Workbox).
- **تخزين مؤقّت** للأصول والخطوط والصور والصوت (runtime caching).
- **عمل دون اتصال أولاً (offline-first)**: كل البيانات محفوظة محلياً على الجهاز.
- واجهة **RTL** عربية بالكامل، وتصميم متجاوب للتابلت أولاً.

### أين تُحفظ البيانات؟
- ملفات الأطفال والتقدّم والإعدادات: `localStorage` (عبر Zustand persist).
- تسجيلات صوت الطفل: `IndexedDB` (مناسب للملفات الصوتية).
- نسخة احتياطية يدوية: تبويب **البيانات → تصدير / استيراد** (ملف JSON).

## نطاق هذه النسخة (MVP)

مكتمل: اختيار الطفل · كلمات مفردة · جملة «أبغى + كلمة» · المشاعر · الاحتياجات الجسدية · الألوان والأشكال · تدريب حرف **ش** (شكل الفم + أمثلة) · تسجيل صوت الطفل وتقييم المعلم (ممتاز/جيد/يحتاج إعادة) · نجوم وتصفيق · التهدئة والتنفّس · تقرير تقدّم · إعدادات حسّية (صوت/اهتزاز/حركة/حجم خط/عدد بطاقات) · قفل المعلم · نسخ احتياطي.

«قريباً» (ظاهرة في الواجهة كرؤية مستقبلية): القصص الاجتماعية · الروتين اليومي · الألعاب التعليمية.

### ملاحظة عن الصور
الصور الحالية رموز تعبيرية (emoji) كعناصر **بديلة** ليعمل التطبيق بلا أصول. استبدلها لاحقاً بـ«مكتبة الصور الآمنة» أو صور الطفل الشخصية: غيّر الحقل `image` في `src/data/words.ts` إلى مسار صورة مستوردة أو `data:`/رابط.

## الجاهزية المستقبلية

- **المزامنة السحابية:** الحالة منفصلة ومُسلسلة (JSON) في `store/` مع دوال `replaceAll` و`exportAll/importBackup` في `lib/backup.ts` — نقطة ربط جاهزة لمزامنة لاحقة (مثل Vercel + Upstash/Redis).

## المستويات 2–6 ونظام اللهجات (v0.2.0)

أُضيفت خمس شاشات مستقلة (لا تُدمج في صفحة واحدة) مطابقة للتصاميم:

- **المستوى الثاني** (بنفسجي): جمل من كلمتين «أبغى + كلمة» — مؤشر خطوات + شبكة كلمات + منطقة جملة.
- **المستوى الثالث** (أخضر): جمل أطول — بطاقات أفقية مع مفضلة + بطاقة تقدّم داخل الشاشة.
- **المستوى الرابع** (وردي): مشاعر / ألم / تهدئة (تبويبات). بطاقات التهدئة بتحفيز هادئ بلا تصفيق مرتفع.
- **المستوى الخامس** (أزرق): ألوان / أشكال / تدريب الجملة. الألوان والأشكال مرسومة بالـ CSS بلا أصول.
- **المستوى السادس** (ذهبي): عبارات يومية (تحية/وداع/أكل/طلب/ردود) + فلتر صعوبة + بطاقة إرشادات.

التنقل بين المستويات عبر «السابق / الرئيسية / التالي»، وكل شاشة فيها: الشريط العلوي (رئيسية + عنوان + وصف + نجوم)، منطقة تدريب (اسمع / سجل / استمع / صحيح / إعادة)، تحفيز بسيط + اهتزاز، واحترام إعدادات الحساسية، والعبارة الرسمية.

### نظام اللهجات (مركزي)
كل النصوص التعليمية تُبنى ديناميكياً من `src/data/dialects.ts` — **لا توجد جمل عربية ثابتة داخل الشاشات**. المفاتيح الثابتة (`water, milk, juice, bathroom, eat, play, sleep, goOut, chips` ...) تُترجم حسب لهجة الطفل وجنسه عبر دوال: `getRequestWord`, `getWordLabel`, `buildRequestPhrase`, `buildEmotionPhrase`, `buildPainPhrase`, `buildColorSentence`, `buildDailyPhrase`، و`profileOf` لاشتقاق ملف الطفل.

- اللهجات: حجازية / نجدية / شرقية / فصحى / مصرية (ولد/بنت) / شامية / عراقية / مخصّصة.
- اللهجة والجنس **لكل طفل** (تُضبط من إعدادات المعلم)، مع لهجة افتراضية للأطفال الجدد.
- تغيير لهجة الطفل يُحدّث كل المستويات تلقائياً — بطاقة الماء مثلاً: «أبغى مويه / أبي ماي / أريد ماء / عايز مية / بدي مي…».
- واجهة الطفل لا تُظهر اسم اللهجة إطلاقاً.

### الإعدادات والتقارير
- إعدادات المعلم: قسم **إعدادات الطفل** (اللهجة + الجنس) وقسم **العبارات اليومية** (مستوى الصعوبة + تشغيل/إيقاف العبارات الدينية).
- التقارير: تفصيل لكل مستوى (محاولات / نجاحات / متقنة / تحتاج تدريب / نسبة) + خاص بالمستوى السادس (أكثر عبارة استخداماً، آخر عبارة، مستوى الصعوبة).

> الصور لا تزال رموزاً تعبيرية/أشكال CSS كبدائل آمنة بلا أصول؛ تُستبدل لاحقاً بمكتبة الصور.

## الحيوانات والألعاب (v0.3.0)

أُضيف قسمان جديدان دون المساس بالمستويات السابقة:

**حيوانات وأصوات** (`AnimalsScreen`): شبكة بطاقات (قط، كلب، بقرة، خروف، حصان، دجاجة، بطة، عصفور، أسد، فيل) — كل بطاقة بزرّين: «اسمع الاسم» و«صوت الحيوان». منطقة تدريب بأزرار: اسمع الاسم / صوت الحيوان / سجل / استمع / صحيح / إعادة. أسماء الحيوانات تأتي من `getAnimalLabel` (قابلة للتخصيص لاحقاً)، وصوت الحيوان من `getAnimalSound`. يوجد إعداد لإيقاف أصوات الحيوانات، وتُوقف تلقائياً في الوضع الحسّاس.

**ألعاب تعليمية** (`GamesScreen` + `MatchingGameScreen`): لعبة مطابقة/ذاكرة بنوعين **مطابقة الألوان** و**مطابقة الحيوانات** (وبطاقتا «قريباً»: صورة+كلمة، صوت+صورة). تدفّق اللعبة: إعداد (٤ مستويات صعوبة: ٤/٦/٨/١٢ كرت + خيارات: مدة الظهور، الحركة، التصفيق، أصوات الحيوانات) ← معاينة (تظهر الكروت ثم تُقلب) ← لعب (اختر كرتين) ← إنهاء. عند التطابق: نطق الاسم + (صوت الحيوان إن فُعّل) + تصفيق خفيف + نجمة + اهتزاز خفيف، وتبقى الكروت مكشوفة. عند الخطأ: «حاول» بلا صوت مزعج، ثم تُقلب الكروت. تستخدم اللعبة **نفس صور الدروس** (ألوان المستوى الخامس + حيوانات قسم الحيوانات).

القواعد الحسّية مطبّقة على الاثنين: في الوضع الحسّاس تُوقف أصوات الحيوانات والتصفيق وحركة الكروت. التقدّم محفوظ في LocalStorage (`kalimati.games.v1` للألعاب، وعدّادات الحيوانات ضمن تقدّم الطفل).

**الإعدادات:** قسم «الألعاب التعليمية» (تشغيل الألعاب، أصوات الحيوانات، عدد الكروت الافتراضي، مدة ظهور الكروت).
**التقارير:** قسم الحيوانات (متقنة، تحتاج تدريب، عدد تشغيل الأصوات، أكثر/آخر حيوان) وقسم الألعاب (ألعاب لُعبت، مكتملة، أفضل نتيجة، أكثر لعبة مفضّلة، آخر جلسة).
**الصفحة الرئيسية:** بطاقات المستويات المرقّمة (٢–٦) + بطاقتا «حيوانات وأصوات» و«ألعاب تعليمية».

## طبقة الوسائط الموحّدة + عرض الألوان (v0.3.2)

أُضيف مصدر صور واحد مركزي `src/data/media.ts` (سجلّ `MEDIA`) يربط كل عنصر بمفتاح ثابت (`cat`, `dog`, `red`, `blue`, `circle`, `water` ...) مع: رمز تعبيري مؤقّت (`emoji`)، ولون (`hex`)، وحقل `image` (حالياً `null`) جاهز لاستبدال الرموز بصور فعلية لاحقاً:

```ts
import catImg from '@/assets/animals/cat.png'
MEDIA.cat.image = catImg   // يُستخدم تلقائياً في الدرس واللعبة معاً
```

كل العناصر تُرسم عبر `mediaVisual(key, size, colorMode)` في `src/components/Media.tsx`، ويقرأ منها **الدرس واللعبة من نفس المصدر** — فلا يختلف شكل العنصر بين الدرس واللعبة. الحيوانات (قسم الحيوانات + لعبة الحيوانات) والألوان/الأشكال (المستوى الخامس + لعبة الألوان) كلّها تقرأ من `media`.

**طريقة عرض الألوان** (إعداد جديد في «الألعاب التعليمية»): `نفس الدرس` (الافتراضي = بقع ألوان مطابقة للمستوى الخامس) / `مربع` / `دائرة`. يؤثّر على عرض الألوان داخل اللعبة فقط؛ الدروس تبقى كما هي.

**القاعدة:** الدرس = نفس الصورة في اللعبة = نفس الصوت = نفس الكلمة حسب اللهجة. النص والصورة غير مكتوبَين داخل الشاشة (يأتيان من `media` + `dialects`).

**الحساسية الحسية** تُعطِّل: أصوات الحيوانات التلقائية، التصفيق العالي، حركة قلب الكروت، والاهتزاز القوي. أزرار واجهة الطفل قصيرة (اسمع، صوته، سجل، استمع، صحيح، إعادة).

## نظام الصور الحقيقية (v0.3.3) — تقرير

النظام الآن **قائم على ملفات صور فعلية** (PNG/WebP/SVG) والرموز التعبيرية مجرد بديل مؤقت.

**أين سجلّ الصور؟** `src/assets/assetRegistry.ts` — يكتشف تلقائياً كل صورة تحت `src/assets/images/**` ويربطها بمفتاحها (اسم الملف). الدالة `getAsset(key)` ترجع رابط الصورة أو `null`.

**أين أضع الصور؟**
- الحيوانات: `src/assets/images/animals/` (cat.png, dog.png, cow.png, …)
- الألوان: `src/assets/images/colors/` (red.png, blue.png, green.png, …)
- وكذلك: `shapes/`, `foods/` (كلمات الطعام/الطلب)، `emotions/`, `daily-phrases/`, `characters/`.
- القاعدة: **اسم الملف = المفتاح الداخلي**، والامتداد png/webp/svg/jpg.

**كيف أستبدل الرموز التعبيرية بالصور؟** فقط ضع الملف باسمه الصحيح في المجلد المناسب (مثل `animals/cat.png`) — لا تعديل على أي شاشة. عند البناء يلتقطه `assetRegistry` ويعرضه `mediaVisual` بدل الرمز. (جرّبتُه فعلياً: وضع `cat.svg` ظهر مباشرة في الحزمة.)

**ترتيب الأولوية في العرض** (`src/components/Media.tsx` → `mediaVisual`):
1. صورة فعلية من `assetRegistry` (أو تجاوز يدوي عبر `MEDIA[key].image`).
2. وإلا بديل مؤقت: بقعة لون (ألوان) / شكل CSS (أشكال) / رمز تعبيري (حيوانات/كلمات).

**هل الدروس والألعاب تقرأ من نفس المصدر؟** نعم. صورة القط في «حيوانات وأصوات» = نفس صورة القط في «مطابقة الحيوانات»، وصورة الأحمر في المستوى الخامس = نفس الأحمر في «مطابقة الألوان» — لأن الجميع يمرّ عبر `mediaVisual` → `assetRegistry`/`media` بنفس المفتاح. والاسم والصوت من `dialects` حسب اللهجة.

> لم يتغيّر تصميم أي شاشة؛ التعديل فقط في بنية الصور. الصيغ المدعومة: png/webp/svg/jpg، والصور الصغيرة تُضمّن تلقائياً (inline) والكبيرة تُحفظ كملفات assets.

## نظام الأصول المركزي (v0.3.4) — التقرير

أصبح للتطبيق **نظام أصول كامل قائم على manifest**، والصور تُدار من مكان واحد وتُستخدم بالمفتاح في كل مكان.

### أين الـ manifest؟
`src/data/assetManifest.ts` (مُولّد آلياً). يربط كل `itemKey` بـ:
```ts
cat: { imageUrl: "/assets/cards/animals/cat.png", category: "animals", sourceType: "animal" }
red: { imageUrl: "/assets/cards/colors/red.png",  category: "colors",  sourceType: "color"  }
```
ودوال: `getAssetUrl(key)` / `hasManifestAsset(key)` / `manifestKeys()`.

### أين أضع الصور؟  (المجلدات جاهزة في `public/assets/`)
- الحيوانات: `public/assets/cards/animals/` (cat.png …)
- الألوان: `public/assets/cards/colors/` (red.png …)
- الأشكال: `shapes/` — المشروبات: `drinks/` — الأكل: `food/` — الأفعال/الاحتياجات والتهدئة: `actions/` — الأشخاص: `people/` — المشاعر والألم: `emotions/` — العبارات اليومية: `daily-phrases/`
- وأيضاً: `icons/`, `rewards/`, `game-cards/`.
- **القاعدة: اسم الملف = المفتاح** (الأسماء المركّبة بشرطة: go-out.png, stomach-pain.png، والمفتاح يبقى goOut/stomachPain).

### كيف أستبدل البدائل بالصور الحقيقية؟
1. الأسهل: ضع ملفاً باسم المفتاح في المجلد الصحيح (مثل `cards/animals/cat.png`) — يستبدل البديل فوراً، **بدون تعديل أي شاشة**.
2. من شيت كبير: ضع الشيت في `tools/extract-assets/sheets/`، عرّف الاقتصاص في `asset-crops.json`، ثم: `npm i -D sharp tsx && npx tsx tools/extract-assets/extractAssets.ts`.
3. مفتاح جديد كلياً: سجّله مرّة واحدة في `assetManifest.ts` فقط.

### هل الدروس والألعاب تقرأ من نفس المصدر؟  نعم.
كل الشاشات تمرّ عبر `mediaVisual(key)` → `assetManifest[key].imageUrl`. لذلك صورة القط في «الحيوانات» = نفسها في «مطابقة الحيوانات» = نفسها في التقرير، والأحمر في «المستوى الخامس» = نفسه في «مطابقة الألوان». النص يأتي من `dialects` حسب اللهجة — **الصورة ثابتة، النص متغيّر**، ولا نص عربي مطبوع داخل الصورة.

### الشاشات المرتبطة بالـ manifest
المستوى الثاني/الثالث/الرابع/الخامس/السادس، الحيوانات، لعبة المطابقة، والتقارير.
> الاستثناء الوحيد: شاشة «كلمات مفردة» (المستوى الأول القديم) ما زالت تستخدم رموزها المحلية؛ يمكن نقلها لاحقاً بنفس الطريقة دون تغيير تصميمها.

### أداة التوليد
`python3 tools/extract-assets/generate_assets.py` تُنشئ المجلدات + البدائل + الـ manifest. (80 أصلاً مولّداً حالياً.)

### صور تحتاج فناً حقيقياً (البدائل الحالية مؤقتة وليست Emoji)
كل البطاقات المرسومة: **الحيوانات (10)، المشروبات (3)، الأكل (4)، الأفعال/الاحتياجات/التهدئة (13)، الأشخاص (2)، المشاعر (6)، الألم (6)، العبارات اليومية (18)**.
البدائل الحالية: الألوان = مربّع لون مصمت، الأشكال = الشكل مرسوماً (مقبولة كنهائية اختيارياً)، والباقي = بطاقة بلون التصنيف برمز «صورة» محايد.

## نظام الأصول الكامل عبر manifest (v0.3.4) — تقرير

تحوّل النظام إلى **manifest مركزي + مجلدات أصول واضحة**، والصور placeholders حقيقية (PNG، بلا نص عربي) قابلة للاستبدال دفعة واحدة دون لمس الشاشات.

**أين الـ manifest؟** `src/data/assetManifest.ts` (مولّد آلياً، 80 مفتاحاً) — يربط كل `itemKey` بـ `{ imageUrl, category, sourceType }`، مع `getAssetUrl(key)` و`hasManifestAsset(key)`. (يبقى `src/assets/assetRegistry.ts` مصدراً ثانوياً اختيارياً.)

**أين أضع صور الحيوانات؟** `public/assets/cards/animals/` بأسماء: `cat.png, dog.png, cow.png, sheep.png, horse.png, chicken.png, duck.png, bird.png, lion.png, elephant.png`.

**أين أضع صور الألوان؟** `public/assets/cards/colors/` بأسماء: `red.png, blue.png, … purple.png`. والأشكال في `public/assets/cards/shapes/`.

**بقية المجلدات:** `drinks/ food/ emotions/ (يشمل الألم) daily-phrases/ people/ actions/`، إضافة إلى `icons/ rewards/ game-cards/` (فارغة جاهزة).

**كيف أستبدل الـ placeholder بصورة فعلية؟** ضع ملفاً فعلياً بنفس المسار/الاسم الموجود في الـ manifest (مثل `public/assets/cards/animals/cat.png`). يلتقطه التطبيق فوراً — **بلا أي تعديل على الشاشات**. أو استخدم أداة القص `tools/extract-assets/extractAssets.ts` مع شيت فعلي.

**هل الدروس والألعاب تقرأ من نفس المصدر؟ نعم.** كل الشاشات تستدعي `mediaVisual(key)` الذي يقرأ من `assetManifest` بنفس المفتاح:
- `cat` → نفس الصورة في «حيوانات وأصوات» + «مطابقة الحيوانات» + التقارير.
- `red` → نفس الصورة في «المستوى الخامس» + «مطابقة الألوان».
- النص والصوت من `dialects.ts` حسب اللهجة (الصورة ثابتة، النص متغيّر).

**ترتيب العرض** في `mediaVisual`: ① صورة الـ manifest → ② تجاوز يدوي `MEDIA[key].image` → ③ `assetRegistry` → ④ بديل CSS (لون/شكل/رمز).

**الأدوات** في `tools/extract-assets/`: `generate_assets.py` (توليد placeholders + manifest)، `extractAssets.ts` + `asset-crops.json` (قص من شيت فعلي)، و`assets.config.json`.

**أصول تحتاج فناً حقيقياً لاحقاً:** كل البطاقات المرسومة (حيوانات/طعام/مشروبات/مشاعر/ألم/أشخاص/أفعال/عبارات يومية) — حالياً placeholders. الألوان/الأشكال نظيفة كما هي. لا حاجة لإضافة أي صورة يدوياً داخل الكود؛ التسجيل مرة واحدة في الـ manifest فقط.
